### AIHUB 可借鉴的核心

1. **等级即权限** — 现有用户角色只有 USER/ADMIN，中间空白太大，需要 Lv1~LvX 等级体系
2. **积分即激励** — 签到、分享、评论、点赞都应产生积分回报，消耗积分可解锁特殊功能
3. **通知即生命线** — 没有通知的社区就像没有门铃的房子，必须作为第一优先级
4. **板块即结构** — 生活圈内应细分话题板块，而不是一个大杂烩

## 3. 第一阶段：轻量增强（2-3天）

> **目标**：用最小改动快速提升社区体验，建立用户反馈通道

### 3.1 🔔 通知系统（最高优先级）

**现状痛点**：用户发了分享、收到评论/点赞，完全没有任何提示

**实施方案**：

**A. 数据库新增**
```prisma
model Notification {
  id        Int      @id @default(autoincrement())
  userId    Int      // 接收通知的用户
  type      String   // 'like' | 'comment' | 'reply' | 'follow' | 'system' | 'mention'
  content   String   // 通知内容摘要
  link      String?  // 跳转链接（如 /user-share?tab=tool#share-123）
  actorId   Int?     // 触发通知的用户ID
  read      Boolean  @default(false)
  createdAt DateTime @default(now())

  @@index([userId, read, createdAt])
  @@map("notifications")
}
```

**B. API 端点**
- `POST /api/notifications` — 创建通知（内部调用）
- `GET /api/notifications?userId=X&unread=true` — 获取通知列表
- `PUT /api/notifications/read` — 标记已读
- `GET /api/notifications/unread-count?userId=X` — 未读数量（Navbar 红点）

**C. 触发点（在现有 API 中嵌入）**
- 评论/回复时 → 给被评论者和楼主的通知
- 点赞分享/评论时 → 给内容作者的通知
- 关注用户时 → 给被关注者的通知
- 系统公告 → 全局通知

**D. 前端改动**
- Navbar 右侧添加铃铛图标 + 未读红点
- 下拉通知面板（最近5条 + 查看全部）
- 通知页面 `/notifications`（全量历史）

**工作量评估**：
- 后端：新增模型 + 2个 API 文件 ≈ 半天
- 前端：通知组件 + 铃铛图标 ≈ 半天
- 嵌入触发：在现有 API 中加入通知创建逻辑 ≈ 半天

### 3.2 🏷️ 话题标签系统

**现状痛点**：热门话题硬编码在侧边栏，分享无标签分类

**实施方案**：
- 在 `Share` 模型新增 `tags` 字段（逗号分隔，类似 Tool 的 tags）
- 发布分享时支持添加 1-3 个标签（#AI工具推荐 #使用心得 #效率提升）
- 侧边栏「热门话题」改为从数据库动态拉取
- 标签自动聚合：`GET /api/tags/popular` 取最近7天使用最多的标签

**工作量**：约半天

### 3.3 📊 用户中心增强

**现状**：用户中心已有 Tab 切换（收藏/点赞/分享），但缺少概览面板

**增强点**：
- 个人主页 `/u/[id]` 完善：显示用户头像、签名、统计（分享数/获赞数/关注/粉丝）
- 添加「最近动态」时间线（发表的分享、评论、点赞）
- 用户中心增加「我的评论」列表

**工作量**：约半天

### 第一阶段总计：约 3 天

## 4. 第二阶段：核心社区化（1周）

> **目标**：建立完整的积分-等级-激励体系，让用户"玩"起来

### 4.1 ⭐ 用户等级 + 积分系统

**数据库新增**：
```prisma
model UserStats {
  id            Int    @id @default(autoincrement())
  userId        Int    @unique
  level         Int    @default(1)   // 1~10 级
  experience    Int    @default(0)   // 当前经验
  expToNext     Int    @default(100) // 升下级所需经验
  contribution  Int    @default(0)   // 贡献积分（可消耗）
  coins         Int    @default(0)   // 金币（可消耗）
  postCount     Int    @default(0)
  commentCount  Int    @default(0)
  likeReceived  Int    @default(0)   // 收到的赞
  likeGiven     Int    @default(0)
  shareCount    Int    @default(0)
  checkinDays   Int    @default(0)

  @@map("user_stats")
}
```

**等级设计**（赛博朋克风格命名）：

| 等级 | 称号 | 所需经验 | 解锁权限 |
|------|------|---------|---------|
| Lv.1 | 电路新手 | 0 | 基础功能 |
| Lv.2 | 代码学徒 | 100 | 自定义头衔 |
| Lv.3 | 黑客实习生 | 300 | 发链接无需审核 |
| Lv.4 | 脚本小子 | 600 | 可上传头像 |
| Lv.5 | 数据行者 | 1000 | 可发视频分享 |
| Lv.6 | 赛博游侠 | 1800 | 自定义签名 |
| Lv.7 | 核心贡献者 | 3000 | 优先审核 |
| Lv.8 | 系统管理员(非) | 5000 | 特殊徽章 |
| Lv.9 | 网络幽灵 | 8000 | 全站高亮标识 |
| Lv.10 | 奇点先知 | 12000 | 所有特权 |

**经验获取规则**：

| 行为 | 经验 | 贡献积分 |
|------|------|---------|
| 每日签到 | +10 | +5 |
| 连续签到(第3天起) | +15/天 | +5 |
| 发布分享 | +20 | +10 |
| 发表评论 | +5 | +3 |
| 收到点赞 | +2 | - |
| 分享被收藏 | +5 | +3 |
| 内容被置顶 | +50 | +30 |
| 举报有效 | +10 | +15 |

**API 变更**：
- `POST /api/user/checkin` — 每日签到（返回经验/连续天数）
- `GET /api/user/stats/:id` — 用户统计
- 在现有的评论/点赞/发布 API 中嵌入经验计算逻辑

### 4.3 📋 板块细分

**现状**：生活圈只有一个大分类，缺乏内容组织结构

**方案 1（轻量）**：给分享添加「子分类」字段
- 在 Share 模型加 `subCategory` 字段
- 生活圈增加子分类 Tab：`全部 | 技术分享 | 效率工具 | 学习资源 | 生活杂谈 | 问答求助`
- 侧边栏可筛选子分类

**方案 2（重量）**：独立「话题」板块
- 新增 `Topic` 模型（类似 Reddit/Linux Do 的帖子）
- 支持富文本内容（Markdown）
- 支持投票、精华标记
- 相当于在现有分享系统上做加法

**推荐**：第一阶段用方案 1，后续升级到方案 2

### 4.4 🔥 动态排行榜

- 日榜：每日点赞最多的分享
- 周榜：本周活跃用户（经验获取量）
- 工具榜：按工具被分享/收藏数量排名
- 贡献榜：按贡献积分排名

**位置**：导航栏「趋势榜」下拉二级菜单增加「社区排行榜」，或在用户分享页侧边栏展示

### 第二阶段总计：约 1 周

## 5. 第三阶段：深度社交+游戏化（2周）

> **目标**：引入私信、圈子、进阶游戏化，打造社区粘性

### 5.1 💬 私信系统

**数据库**：
```prisma
model Conversation {
  id        Int      @id @default(autoincrement())
  title     String?  // 群聊名称（可选）
  type      String   @default("direct") // direct | group
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
  participants ConversationParticipant[]
  messages    Message[]

  @@map("conversations")
}

model ConversationParticipant {
  id             Int      @id @default(autoincrement())
  conversationId Int
  userId         Int
  lastReadAt     DateTime @default(now())
  joinedAt       DateTime @default(now())

  @@unique([conversationId, userId])
  @@map("conversation_participants")
}

model Message {
  id             Int      @id @default(autoincrement())
  conversationId Int
  senderId       Int
  content        String
  createdAt      DateTime @default(now())

  @@index([conversationId, createdAt])
  @@map("messages")
}
```

**功能**：
- 用户主页增加「发私信」按钮
- 私信列表 `/messages`（类似微信聊天列表）
- 实时消息（WebSocket / 轮询）
- 私信通知（未读消息数 + 通知红点）

### 5.2 👥 用户圈子/小组

- 用户可以创建「圈子」（兴趣小组）
- 每个圈子有独立的话题墙
- 类似：AI 绘画交流圈、Prompt 技巧圈、开源项目圈
- 圈主可管理成员/精华帖

**数据库**：
```prisma
model Circle {
  id          Int      @id @default(autoincrement())
  name        String   @unique
  description String
  icon        String?
  bannerUrl   String?
  creatorId   Int
  memberCount Int      @default(1)
  postCount   Int      @default(0)
  isPublic    Boolean  @default(true)
  createdAt   DateTime @default(now())

  @@map("circles")
}

model CircleMember {
  id        Int      @id @default(autoincrement())
  circleId  Int
  userId    Int
  role      String   @default("MEMBER") // MEMBER | MOD | OWNER
  joinedAt  DateTime @default(now())

  @@unique([circleId, userId])
  @@map("circle_members")
}
```

### 5.3 🎮 进阶游戏化

- **宠物系统深度联动**：分享/评论获得宠物饲料，宠物等级影响用户签到加成
- **道具系统**：积分可购买「置顶卡」「高亮卡」「改名卡」
- **每日任务**：系统派发每日任务（今日分享3条、评论5条等），完成获额外经验
- **连续签到奖励轮**：7天/14天/30天里程碑奖励

### 5.4 📧 邮件通知增强

**现状**：用户表已有 `notifyComment` / `notifyLike` / `notifyFollow` / `notifyEmail` 字段，但未实现

**实现**：
- 将通知系统的关键事件（被回复、被点赞）通过邮件发送
- 邮件模板：简洁的 HTML 风格，与赛博朋克主题一致
- 频率控制：同事件15分钟内不重复发送

### 第三阶段总计：约 2 周

```
第 1-3 天  →  通知系统 + 标签系统 + 用户主页增强
第 4-7 天  →  积分等级 + 成就徽章
第 8-10 天 →  板块细分 + 排行榜
第 11-14天 →  私信系统（可选）
第 15天+   →  兴趣圈子 / 专栏系统（视情况）

总工期：约 2-3 周可完成核心社区化改造
```

### 💡 我的建议

**阿童，如果时间有限，我建议这样切：**

1. **先做通知系统** — 这是社区的"门铃"，不做通知用户永远不知道有人回复/点赞了
2. **然后做积分等级** — 让用户有成长感，签到+积分是最快提升留存的手段
3. **最后做板块细分** — 在生活圈内加子分类，让内容有组织

这三步走完，你的社区基本就"活"了。剩下的私信、圈子、专栏都是锦上添花，可以等用户量上来再按需开发。
